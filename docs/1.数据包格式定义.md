# 数据包格式定义 (Knowledge Package Specification)

ICE (Industry Chain Explorer) v2.0 采用**知识中心架构**，所有知识以结构化数据包形式存储，分为三层：
- **Ontology（本体层）**：定义分类体系和谓词（关系类型）
- **Knowledge（知识层）**：存储对象(Object)和事实(Fact)
- **Package（产业包层）**：定义产业视图和分层布局

---

## 目录结构

```
knowledge-root/
├── ontology/              # 本体层（全局共享）
│   ├── taxonomy.yaml      # 分类体系
│   └── predicates.yaml    # 谓词（关系类型）定义
├── knowledge/             # 知识层
│   ├── objects/           # 对象（每个文件一个对象）
│   │   └── *.yaml
│   ├── facts/             # 事实（每个文件一个三元组关系）
│   │   └── fact_*.yaml
│   └── references/        # 引用文档（可选）
│       └── *.yaml
└── packages/              # 产业包层
    └── {package_id}/      # 每个产业一个目录
        ├── package.yaml   # 产业包元信息
        └── layout.yaml    # 布局配置（可选）
```

## 1. Ontology 本体层

### 1.1 taxonomy.yaml - 分类体系

定义所有概念/组织/文献的分类层级，支持多级继承。

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `categories` | dict | ✅ | 分类字典，key为分类ID |
| `categories.{id}.name` | string | ✅ | 分类中文名称 |
| `categories.{id}.parent` | string/null | ✅ | 父分类ID，根分类为null |

**示例：**
```yaml
categories:
  industry:
    name: 产业
    parent: null
  communication:
    name: 通信
    parent: industry
  optical_communication:
    name: 光通信
    parent: communication
  product:
    name: 产品
    parent: null
  optical_module:
    name: 光模块
    parent: product
```

**标准分类根节点：**
- `industry` - 产业
- `technology` - 技术
- `component` - 器件
- `material` - 材料
- `product` - 产品
- `application` - 应用
- `organization_type` - 组织类型（company/research_institute/standard_organization）
- `document_type` - 文献类型（research_report/annual_report/standard/white_paper）

---

### 1.2 predicates.yaml - 谓词（关系类型）

定义对象之间可以存在的关系类型，约束主语和宾语类型，支持定义反向关系。

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `predicates` | dict | ✅ | 谓词字典，key为谓词ID |
| `predicates.{id}.name` | string | ✅ | 谓词中文名称 |
| `predicates.{id}.subject` | list[string] | ✅ | 允许的主语类型：concept/organization/document |
| `predicates.{id}.object` | list[string] | ✅ | 允许的宾语类型：concept/organization/document |
| `predicates.{id}.inverse` | string | ❌ | 反向谓词ID（双向关系自动推导） |
| `predicates.{id}.symmetric` | bool | ❌ | 是否对称关系（默认false），对称关系主语宾语可互换 |

**内置谓词：**

| 谓词ID | 中文名 | 说明 | 反向 |
|--------|--------|------|------|
| `is_a` | 属于 | 分类关系，A是B的一种 | `has_instance` |
| `part_of` | 是...的一部分 | 组成关系 | `contains` |
| `produce` | 生产 | 组织生产产品 | `produced_by` |
| `supply` | 供应 | 供应链关系 | `supplied_by` |
| `use` | 使用 | 使用关系 | `used_by` |
| `apply_to` | 应用于 | 技术应用于产品 | `applies` |
| `replace` | 替代 | 技术/产品替代 | `replaced_by` |
| `compete_with` | 与...竞争 | 竞争关系（对称） | - |
| `depends_on` | 依赖 | 依赖关系 | - |
| `defined_by` | 由...定义 | 概念由标准/文献定义 | - |
| `published_by` | 由...发布 | 文献由组织发布 | - |
| `research_on` | 研究 | 组织研究某领域 | - |
| `related_to` | 相关 | 通用关联关系（对称） | - |
| `integrate` | 集成 | 集成关系 | `integrated_in` |

**示例：**
```yaml
predicates:
  is_a:
    name: 属于
    subject: [concept, organization, document]
    object: [concept]
    inverse: has_instance
  supply:
    name: 供应
    subject: [concept, organization]
    object: [concept, organization]
    inverse: supplied_by
  compete_with:
    name: 与...竞争
    subject: [concept, organization]
    object: [concept, organization]
    symmetric: true
```

## 2. Knowledge 知识层

### 2.1 Object 对象

每个YAML文件定义一个知识对象，文件名即为对象ID（推荐）。

**通用字段（所有类型）：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 对象ID，格式：`^[a-z][a-z0-9_]*$`（小写字母开头，仅小写/数字/下划线） |
| `kind` | enum | ✅ | 对象类型：`concept`（概念）/ `organization`（组织）/ `document`（文献） |
| `name` | string | ✅ | 对象中文名称 |
| `aliases` | list[string] | ❌ | 别名/英文名/缩写，用于搜索匹配 |
| `summary` | string | ❌ | 一句话摘要 |
| `tags` | list[string] | ❌ | 标签 |
| `deprecated_ids` | list[string] | ❌ | 废弃的旧ID，用于兼容历史数据 |
| `created_at` | date | ❌ | 创建日期，格式：YYYY-MM-DD |
| `updated_at` | date | ❌ | 更新日期，格式：YYYY-MM-DD |

**concept（概念）类型：** 无额外必填字段。

**organization（组织）类型额外字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `organization_type` | string | ❌ | 组织类型：company/research_institute/standard_organization |
| `code` | string | ❌ | 股票代码 |
| `market` | string | ❌ | 上市市场：SH/SZ/HK/NYSE/NASDAQ |

**document（文献）类型额外字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `document_type` | string | ❌ | 文献类型：research_report/annual_report/standard/white_paper |
| `title` | string | ❌ | 文献完整标题 |
| `author` | string | ❌ | 作者/发布机构 |
| `published_date` | string | ❌ | 发布日期，格式：YYYY或YYYY-MM-DD |
| `source_url` | string | ❌ | 来源URL |

**示例 - 概念：**
```yaml
id: optical_module
kind: concept
name: 光模块
aliases:
  - Optical Module
  - 光收发模块
summary: 光通信核心器件，实现光电信号转换，广泛应用于数据中心和电信网络。
tags: [光模块, 通信, 光通信]
created_at: "2026-01-01"
updated_at: "2026-06-26"
```

**示例 - 组织（公司）：**
```yaml
id: zhongji_innolight
kind: organization
name: 中际旭创
aliases:
  - Innolight
summary: 全球光模块龙头企业，800G/1.6T产品领先。
organization_type: company
code: "300308"
market: SZ
tags: [光模块, 800G, 1.6T, 龙头, 海外]
created_at: "2026-01-01"
updated_at: "2026-06-26"
```

**示例 - 文献（研究报告）：**
```yaml
id: gs_optical_2025
kind: document
name: 国盛证券光模块深度报告
title: 光模块行业深度报告
author: 国盛证券
summary: 覆盖光模块完整价值链的行业深度研究报告。
document_type: research_report
tags: [研究报告, 光模块, 行业分析]
published_date: "2025"
created_at: "2026-01-01"
updated_at: "2026-06-26"
```

### 2.2 Fact 事实（三元组）

每个YAML文件定义一个事实（RDF风格三元组：主语-谓词-宾语），文件名格式：`fact_NNNNN.yaml`（NNNNN为5位数字）。

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 事实ID，格式：`fact_[0-9]+`（如fact_00001） |
| `subject` | string | ✅ | 主语对象ID |
| `predicate` | string | ✅ | 谓词ID（必须是predicates.yaml中定义的） |
| `object` | string | ✅ | 宾语对象ID |
| `statement` | string | ❌ | 事实的自然语言陈述，用于展示 |
| `qualifiers` | dict | ❌ | 限定符/修饰属性（如数值、时间、版本） |
| `citations` | list[string] | ✅ | 引用来源，引用document对象的ID列表（不允许为空） |
| `weight` | float | ❌ | 权重/置信度 0.0-1.0，用于排序 |
| `created_at` | date | ❌ | 创建日期 |
| `updated_at` | date | ❌ | 更新日期 |

**示例：**
```yaml
id: fact_00007
subject: optical_module_800g
predicate: use
object: pam4
statement: 800G光模块采用PAM4调制技术，每通道112Gbps PAM4信号。
qualifiers:
  data_rate: "800G"
  modulation: PAM4
  per_channel: "112Gbps"
citations: [gs_optical_2025, oif_224g]
weight: 1.0
created_at: "2026-01-01"
updated_at: "2026-06-26"
```

**分类关系示例（is_a）：**
```yaml
id: fact_00004
subject: optical_module
predicate: is_a
object: product
statement: 光模块属于光通信产业链核心产品。
citations: [gs_optical_2025]
created_at: "2026-01-01"
updated_at: "2026-06-26"
```

## 3. Package 产业包层

### 3.1 package.yaml - 产业包元信息

定义产业包的基本信息、入口点和分层配置。

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 产业包ID，与目录名一致，格式同Object ID |
| `name` | string | ✅ | 产业包名称 |
| `version` | string | ✅ | 版本号，语义化版本如"2.0.0" |
| `description` | string | ❌ | 产业包描述 |
| `entry_points` | list[string] | ✅ | 入口对象ID列表，用户打开时默认展示这些对象 |
| `default_layout` | string | ❌ | 默认布局：`layered`（分层，默认）/ `graph`（关系图） |
| `layers` | list[Layer] | ✅ | 分层配置，用于Layer View展示 |
| `layers[].id` | string | ✅ | 层ID |
| `layers[].name` | string | ✅ | 层中文名称 |
| `layers[].categories` | list[string] | ✅ | 该层包含的分类ID列表（来自taxonomy） |
| `layers[].order` | int | ✅ | 排序顺序，数字越大越靠上（应用层order最大） |
| `theme` | string | ❌ | 主题色：blue/green/orange/purple |
| `keywords` | list[string] | ❌ | 关键词列表 |

**示例（光模块产业包）：**
```yaml
id: optical
name: 光模块产业链
version: "2.0.0"
description: 光模块是光通信核心器件，实现光电信号转换。本产业包覆盖光模块完整价值链，从上游材料/器件到下游AI服务器/数据中心应用。
entry_points:
  - optical_module
default_layout: layered
theme: blue
keywords:
  - AI
  - 算力
  - 通信
  - 800G
  - 1.6T
  - 光模块
  - 硅光
layers:
  - id: application
    name: 应用层
    categories: [ai_infra, application]
    order: 4
  - id: product
    name: 产品层
    categories: [product]
    order: 3
  - id: component
    name: 器件层
    categories: [component]
    order: 2
  - id: material
    name: 材料层
    categories: [material]
    order: 1
  - id: technology
    name: 技术层
    categories: [technology]
    order: 0
```

## 4. ID 命名规范

| 类型 | 命名规范 | 示例 |
|------|---------|------|
| Object ID | 全小写，下划线分隔，英文单词/缩写 | `optical_module`, `zhongji_innolight`, `gs_optical_2025` |
| Fact ID | `fact_` + 5位数字，从00001开始递增 | `fact_00001`, `fact_00050` |
| Package ID | 全小写，下划线分隔 | `optical`, `robotics`, `semiconductor` |
| Predicate ID | 小写下划线，动词或动词短语 | `is_a`, `supply`, `compete_with` |
| Category ID | 小写下划线 | `optical_module`, `ai_infra`, `research_report` |

## 5. 验证规则

系统加载时自动执行五层验证：

1. **Schema验证**：所有YAML字段符合类型定义，禁止额外字段（extra="forbid"）
2. **引用完整性**：Fact的subject/object/citations必须指向存在的Object
3. **谓词约束**：Fact的主语/宾语类型必须符合predicates.yaml中定义的类型约束
4. **分类验证**：Object的分类必须存在于taxonomy.yaml中
5. **质量检查**：
   - Fact必须至少有一个citation（警告）
   - Object建议设置summary（信息）
   - Object建议设置aliases提高搜索命中率（信息）

**加载模式：**
- `strict=False`（默认）：错误和警告都显示，可继续加载
- `strict=True`：有任何错误则抛出异常终止加载

## 6. 最小示例

最小可运行数据包仅需：
1. `ontology/predicates.yaml` - 至少定义is_a谓词
2. `ontology/taxonomy.yaml` - 至少定义根分类
3. `knowledge/objects/` - 至少2个概念对象
4. `knowledge/facts/` - 至少1条is_a关系
5. `packages/{id}/package.yaml` - 产业包配置
