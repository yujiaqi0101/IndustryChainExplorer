# 规则
## 核心规则：
0.每次回复的时候必须以，余老板：开头；
1.总是使用中文回复所有问题，做事必须实事求是；
2.每个完成的功能都要做好最小单元的测试；
3.使用plan模型生成的计划，写成MD文件存放到/docs/plans目录下；
4.做Spec的时候要理解任务，确认tasks list和check list。
5.重构任务时必须检查数据库结构和新结构是否一致！
6.修复bug时必须先确认任务的根本原因是什么，如果难以确认那么
6.1 首先判断是否可以通过独立脚本验证，测试脚本放到项目根目录下的/tests目录下，如果没有目录则新建，问题修复后要清除；
6.2 其次考虑在现有代码中增加日志判断原因，问题修复后要清除；
